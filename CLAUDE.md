# CLAUDE.md - Claude Code 設定

このファイルは Claude Code の動作を制御する設定ファイルです。

## プロジェクト概要

- **モード**: Solo
- **対象ユーザー**: 個人プロジェクト
- **開発言語**: Python (推定)

## 開発ワークフロー

### 基本サイクル

1. **Plan** (`/plan-with-agent`) - 実装計画の作成
2. **Work** (`/work`) - Plans.md のタスクを実行
3. **Review** (`/harness-review`) - コード品質チェック
4. **Verify** (`/verify`) - ビルド・テスト検証

### Plans.md の管理

- タスクは Plans.md で一元管理
- 状態: `[ ]` (未着手) → `[~]` (進行中) → `[x]` (完了)
- 完了したタスクは定期的にアーカイブ

## ディレクトリ構造

```
.
├── AGENTS.md           # 開発フロー概要
├── CLAUDE.md           # この設定ファイル
├── Plans.md            # タスク管理
├── .claude/
│   ├── settings.json   # 権限・安全設定
│   ├── memory/         # 意思決定記録・パターン
│   └── rules/          # 品質保護ルール
```

## 記憶システム (SSOT)

### decisions.md
- プロジェクトの重要な意思決定を記録
- 「なぜこの技術を選んだのか」などの背景

### patterns.md
- 再利用可能なパターンを記録
- コーディング規約、命名規則など

## 品質保護ルール

### test-quality.md
- テストの改ざん禁止
- テストが失敗したら原因を修正する

### implementation-quality.md
- 形骸化実装の禁止
- 実際に動作するコードを書く

## Skills Gate

このプロジェクトで有効化されているスキル:
- `impl` - 機能実装
- `review` - コードレビュー
- `verify` - ビルド検証
- `troubleshoot` - トラブルシューティング

変更するには: `/skills-update`

## セッション管理

### セッション開始時

```
/sync-status
```

- 前回の続きから始められる
- 進捗を確認して次のアクションを提案

### セッション終了時

- Plans.md が自動的に更新される
- 重要な決定は decisions.md に記録される

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| ビルドが通らない | `/verify` で自動修正 |
| 何をすればいいかわからない | `/sync-status` で状況確認 |
| エラーが出た | 「動かない」「エラーが出た」と伝える |

## VibeCoder モード

技術的な知識がなくても、自然言語で開発できます。

**例**:
- 「ログイン機能を追加して」
- 「動かして」
- 「続けて」
- 「どうすればいい?」

詳しくは AGENTS.md を参照してください。
