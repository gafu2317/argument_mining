# AGENTS.md - エージェント開発フロー

このプロジェクトでは、Claude Code を使った Solo モードの開発フローを採用しています。

## 開発フロー概要

```
┌─────────────────┐
│  1. Plan        │  計画立案
│  /plan-with-agent │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  2. Work        │  実装作業
│  /work          │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. Review      │  コードレビュー
│  /harness-review │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  4. Verify      │  ビルド検証
│  /verify        │
└─────────────────┘
```

## 基本的な使い方

### 1. 計画を立てる

```
/plan-with-agent 〇〇を作りたい
```

- 実装したい機能やタスクを自然言語で伝える
- Plans.md にタスクが自動的に追加される

### 2. 実装する

```
/work
```

- Plans.md のタスクを実行
- 並列実行も自動で最適化

### 3. レビューする

```
/harness-review
```

- セキュリティ、パフォーマンス、品質をチェック
- 指摘事項を確認して改善

### 4. 検証する

```
/verify
```

- ビルド、テスト、lint を実行
- エラーがあれば自動修正

## VibeCoder 向けヒント

| やりたいこと | 言い方 |
|-------------|--------|
| 続きをやる | 「続けて」「次」 |
| 動作確認 | 「動かして」「見せて」 |
| 機能追加 | 「〇〇を追加して」 |
| 困った | 「どうすればいい？」 |
| 全部任せる | 「全部やって」 |

## 進捗確認

```
/sync-status
```

- 現在の進捗を確認
- Plans.md を更新
- 次のアクションを提案

## その他の便利なコマンド

- `/skill-list` - 利用可能なスキル一覧を表示
- `/harness-mem` - セッション記録の管理
- `/harness-update` - ハーネスの最新版へアップデート
